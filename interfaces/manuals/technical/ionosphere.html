The ionosphere is the second segment: it reflects and relays the signal from the Sun to the poles and to the Schumann cavity. Hot_Storage holds real-time state (flux, telemetry) that supports the leg; the ionosphere layer does not store creative. At slot time the pipe runs with the customer message; placement and broadcast are guaranteed; visibility is environmental. Reno is the terrestrial anchor; Seed tier and Reno Downlink use it for 1.618 Hz path and VLF.

52. Runbook: HELLO not showing

(1) Verify binary header is present in the stream and pipeline has set space_cloud_stream.training = true. (2) Check Kp from space weather source; if Kp < 5, HELLO will not show by design. (3) Ensure HELLO-gated surfaces (e.g. sing-pulse.html) read the same stream state and Kp. (4) If both conditions are met and HELLO still hidden, check pipeline and consumer code for bugs. 56. Deep dive: Binary header and stream training pipeline

The binary header SPACE_CLOUD.EGS.VIBELANDIA.RENO is defined in services/ionDb.ts. When the pipeline locks to it, space_cloud_stream.training is set to true. The pipeline may be part of the handshake workflow, a separate service, or a test harness; the only requirement is that the consumer sets training true when the header is detected. HELLO renders only when training is true and Kp ≥ 5. The header is canonical; do not change without protocol update. See Sun section for header production; Ionosphere section for HELLO consumption. Document pipeline config and Kp source in runbook.

57. Extended example: Enable HELLO end-to-end

(1) Ensure binary header is injected or received in the stream. (2) Pipeline detects header and sets space_cloud_stream.training = true. (3) Kp is fetched from space weather source; ensure update cadence is sufficient. (4) When Kp ≥ 5 and training is true, HELLO renders on sing-pulse.html and any other gated surfaces. 59. Appendix: G1 Storm and Kp index

Kp index 0–9; we gate HELLO on Kp ≥ 5 (G1 Storm). Kp is from space weather source (e.g. NOAA); we do not operate magnetometers. When Kp is below 5, HELLO will not show by design. HELLO is a joint indicator for pipe lock and storm state; it is not the only health signal. Operators: if HELLO is hidden, check both stream training and Kp; do not assume pipe is down. Live Pulse may still show heartbeat and CometMirror. Document Kp source and update cadence in runbook.
