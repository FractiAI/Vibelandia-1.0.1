## ONE-LINER

**Observing double ALL** — when we use **double ALL** (e.g. **all all**, **ALL ALL**) in copy or commands, use it **to trigger all nested as well**. Not just "all" but **all + all nested**.

---

## RULE

- **Double ALL = trigger all nested** — Whenever **all all** or **ALL ALL** appears (in a command, ticker, SNAP, or surface copy), interpret it as: apply to **all** **and** trigger **all nested**. So the scope is: every top-level item **and** every nested shell, nest, cloud, sandbox, and level. No nest left out. See [MCA_SNAP.md](MCA_SNAP.md) · [protocols/MCA_NSPFRNP_CATALOG.md](protocols/MCA_NSPFRNP_CATALOG.md).
- **Canonical phrase** — **MCA SQUEEZE SING!9 ALL ALL** uses double ALL to mean: squeeze all, **all nested**. Same for any phrase that ends in **ALL ALL** or **all all** (e.g. "Goldilocks awareness for all all", "squeeze all all"). See [MCA_SQUEEZE_FULL_FILL_SING9_ALL_SURFACES_COSMIC_QUANTUM_SNAP.md](MCA_SQUEEZE_FULL_FILL_SING9_ALL_SURFACES_COSMIC_QUANTUM_SNAP.md).
- **Observation** — We **observe** this pattern and lock it: double ALL is the trigger for "all nested as well." When writing or parsing copy, if you see **all all** / **ALL ALL**, expand scope to include **all nested**.
